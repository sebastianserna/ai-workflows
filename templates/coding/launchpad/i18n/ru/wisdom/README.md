# wisdom/ — База знаний

Эта папка содержит техническую документацию проекта. Она является **источником истины**, с которым ИИ-агенты сверяются перед принятием решений.

## Назначение

- Документировать архитектурные решения и их обоснования
- Поддерживать технические руководства в актуальном состоянии
- Не допускать повторения ошибок или принятия противоречивых решений агентами
- Сохранять знания, которые не помещаются в самом коде

## Структура

```
wisdom/
├── README.md           # Этот файл
├── architecture/       # Как спроектирована система
├── development/        # Как работать над проектом
└── operations/         # Как система развёртывается и эксплуатируется
```

### architecture/

Архитектурные решения, структура системы, выбранные паттерны и их обоснования. Отвечает на вопрос **«как спроектировано и почему»**.

Примеры: структура папок, паттерны слоёв, решения по выбору библиотек, диаграммы компонентов.

### development/

Руководства для повседневной разработки. Отвечает на вопрос **«как работать над этим проектом»**.

Примеры: соглашения по коду, руководство по тестированию, git-процесс, локальная настройка, линтинг.

### operations/

Всё, что связано с доставкой кода в продакшен и поддержкой его работоспособности. Отвечает на вопрос **«как развёртывается и эксплуатируется»**.

Примеры: CI/CD пайплайн, конфигурация окружений, мониторинг, переменные окружения.

## Когда добавлять новые папки

Три базовые папки покрывают большинство проектов. Если проект растёт в определённой области, добавляйте специализированные папки:

| Папка | Когда добавлять |
|-------|-----------------|
| `database/` | Проект имеет собственную базу данных с миграциями |
| `security/` | Правила аутентификации, разрешения или требования соответствия для документирования |
| `integrations/` | Несколько внешних API или сторонних сервисов |
| `api/` | Публичный API, используемый другими |

**Правило:** не создавайте пустые папки «на всякий случай». Создавайте их, когда есть хотя бы один реальный документ для размещения.

## Соглашения

- Документы в формате Markdown, одна тема на файл
- Обновляйте при изменении архитектуры
- Ссылайтесь из `AGENTS.md` в разделе «Подробные спецификации»

## Отличие от workplan/

| | `wisdom/` | `workplan/` |
|---|-----------|-------------|
| **Содержит** | Постоянную документацию | Рабочие планы с жизненным циклом |
| **Изменяется** | При эволюции архитектуры | Постоянно (backlog → coding → done) |
| **Формат** | Свободный | Стандартизированный шаблон с заголовком |
| **Назначение** | Понимание системы | Управление тем, что нужно сделать |
