## फ़ाइल नामकरण प्रारूप

```
TYPE-YYYY-MM-DD-NNNN-description.md
```

| खंड | विवरण | उदाहरण |
|-----|-------|--------|
| `TYPE` | स्थिति: `BACKLOG`, `CODING`, `DONE`, `DRAFT` | `BACKLOG` |
| `YYYY-MM-DD` | वर्तमान स्थिति में संक्रमण की तिथि | `2026-01-15` |
| `NNNN` | आंतरिक अनुक्रमिक ID, 4 अंक शून्य-पैडेड | `0001` |
| `description` | kebab-case में संक्षिप्त नाम | `user-auth-setup` |

### आंतरिक ID (`NNNN`)

प्रोजेक्ट-विशिष्ट वृद्धिशील काउंटर। अगला प्राप्त करने के लिए:

1. सभी सबफ़ोल्डर (`backlog/`, `coding/`, `done/`) में सबसे बड़ी `NNNN` खोजें
2. 1 जोड़ें

ID **स्थायी** है — यह नहीं बदलती भले ही योजना की स्थिति या नाम बदल जाए।

> **आरक्षित:** ID `0000` उदाहरण/टेम्पलेट फ़ाइलों के लिए आरक्षित है। पहली वास्तविक योजना को `0001` का उपयोग करना चाहिए।

### तिथि नियम

फ़ाइल नाम में तिथि वर्तमान स्थिति में संक्रमण को दर्शाती है:

| स्थिति | नाम में तिथि = |
|--------|---------------|
| BACKLOG | योजना बनाने की तिथि |
| CODING | काम शुरू करने की तिथि |
| DONE | पूर्ण होने की तिथि |

## फ़ोल्डर संरचना

```
workplan/
├── README.md          # यह फ़ाइल (प्रामाणिक स्रोत)
├── backlog/           # लंबित योजनाएँ
├── coding/            # प्रगति में कार्य
├── done/              # पूर्ण योजनाएँ (संग्रह)
└── draft/             # प्रारूप, विचार और निर्णय (कोई स्थिति वर्कफ़्लो नहीं)
```

## मानकीकृत हेडर

```markdown
# योजना: वर्णनात्मक शीर्षक

**स्थिति:** Backlog | Coding | Done
**Backlog:** 2026-01-15
**Coding:** —
**Done:** —
**डोमेन:** सामान्य
**Issue:** —
```

**हेडर नियम:**
- स्थिति को फ़ाइल नाम के प्रीफ़िक्स से मेल खाना चाहिए
- तिथियाँ दर्ज करती हैं कि प्रत्येक संक्रमण कब हुआ (`—` यदि लागू नहीं)
- **Issue** `—` है यदि issue ट्रैकर "none" के रूप में कॉन्फ़िगर किया गया है

## मानक टेम्पलेट

### आवश्यक शब्दावली

| शब्द | उपयोग | उदाहरण |
|------|-------|--------|
| **चरण** | प्रमुख कार्यान्वयन मील का पत्थर | चरण 1: MVP, चरण 2: सुधार |
| **कदम** | प्रगति चेकलिस्ट में ठोस आइटम | `- [ ] SQL माइग्रेशन बनाएँ` |

### अनुभाग क्रम

हेडर → प्रगति → उद्देश्य → संदर्भ → कार्यान्वयन → सत्यापन → जोखिम

वैकल्पिक अनुभागों को **छोड़ दिया जाता है**, खाली नहीं छोड़ा जाता।

### टेम्पलेट

```markdown
# योजना: वर्णनात्मक शीर्षक

**स्थिति:** Backlog
**Backlog:** YYYY-MM-DD
**Coding:** —
**Done:** —
**डोमेन:** सामान्य
**Issue:** —

## प्रगति

### चरण 1: MVP
- [ ] ठोस, सत्यापन योग्य कदम
- [ ] अन्य कदम

### चरण 2: सुधार *(यदि लागू हो)*
- [ ] ठोस, सत्यापन योग्य कदम

## उद्देश्य

क्या हासिल करना है और क्यों (1-3 पैराग्राफ)।

## संदर्भ *(वैकल्पिक)*

वर्तमान स्थिति, पूर्वापेक्षाएँ।

## कार्यान्वयन

### चरण 1: MVP

कार्यान्वयन का तकनीकी विवरण।

### चरण 2: सुधार *(वैकल्पिक)*

## सत्यापन *(वैकल्पिक)*

योजना सही ढंग से पूर्ण हुई इसे मान्य करने के कदम।

## जोखिम *(वैकल्पिक)*

केवल जटिल योजनाओं के लिए।
```

### नियम

1. **प्रगति हमेशा हेडर के बाद**, कभी अंत में नहीं
2. कदम चरण के अनुसार समूहित (`### चरण N: नाम`)
3. प्रत्येक कदम ठोस और सत्यापन योग्य होना चाहिए (सामान्य नहीं)
4. तकनीकी विवरण कार्यान्वयन में, सारांश प्रगति में
5. केवल "चरण" और "कदम" का उपयोग करें
6. वैकल्पिक अनुभाग छोड़ दिए जाते हैं, खाली नहीं छोड़े जाते

## वर्कफ़्लो

### योजना बनाएँ

1. अगली अनुक्रमिक ID (`NNNN`) प्राप्त करें
2. `backlog/BACKLOG-YYYY-MM-DD-NNNN-description.md` में फ़ाइल बनाएँ

### काम शुरू करें

1. `backlog/` से `coding/CODING-YYYY-MM-DD-NNNN-description.md` में स्थानांतरित करें (तिथि = आज, ID अपरिवर्तित)
2. हेडर अपडेट करें: `स्थिति: Coding`, `Coding: YYYY-MM-DD`

### पूर्ण करें

1. `coding/` से `done/DONE-YYYY-MM-DD-NNNN-description.md` में स्थानांतरित करें (तिथि = आज)
2. हेडर अपडेट करें: `स्थिति: Done`, `Done: YYYY-MM-DD`

### बैकलॉग में वापस

1. `coding/` से `backlog/BACKLOG-YYYY-MM-DD-NNNN-description.md` में स्थानांतरित करें
2. हेडर अपडेट करें: `स्थिति: Backlog`

## क्रॉस-रेफ़रेंस

योजनाएँ एक-दूसरे को **आंतरिक ID** (`NNNN`) का उपयोग करके संदर्भित करती हैं, जो स्थायी है:

```markdown
**निर्भरताएँ:** योजना 0001, योजना 0002

विवरण के लिए योजना 0003 देखें।
```

**नियम:** किसी योजना को पूर्ण फ़ाइल नाम से कभी संदर्भित न करें (यह प्रत्येक संक्रमण पर बदलता है)। हमेशा आंतरिक ID का उपयोग करें।

## अगली ID प्राप्त करें

```bash
# सभी सबफ़ोल्डर में सबसे बड़ी NNNN खोजें
ls workplan/{backlog,coding,done}/ | grep -oP '\d{4}' | sort -n | tail -1
# परिणाम में 1 जोड़ें
```

## प्रारूप (ड्राफ़्ट)

प्रारूप `draft/` में `DRAFT-YYYY-MM-DD-description.md` प्रारूप में संग्रहीत होते हैं। ये BACKLOG→CODING→DONE वर्कफ़्लो का पालन नहीं करते और इनमें कोई अनुक्रमिक ID नहीं होती। ये विचार बैंक के रूप में काम करते हैं: योजना प्रारूप, तकनीकी निर्णय, खोजपूर्ण नोट। जब कोई प्रारूप पर्याप्त रूप से परिपक्व हो जाता है, तो इसे `backlog/` में औपचारिक योजना के रूप में प्रोन्नत किया जाता है।
