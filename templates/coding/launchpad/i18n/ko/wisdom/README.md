# wisdom/ — 지식 베이스

이 폴더는 프로젝트의 기술 문서를 포함합니다. AI 에이전트가 의사결정 전에 참고하는 **신뢰할 수 있는 정보원**입니다.

## 목적

- 아키텍처 결정과 그 이유를 기록
- 기술 가이드를 최신 상태로 유지
- 에이전트가 같은 실수를 반복하거나 일관성 없는 결정을 내리는 것을 방지
- 코드 자체로 표현할 수 없는 지식을 보존

## 구조

```
wisdom/
├── README.md           # 이 파일
├── architecture/       # 시스템 설계 방식
├── development/        # 프로젝트 작업 방법
└── operations/         # 배포 및 운영 방법
```

### architecture/

아키텍처 결정, 시스템 구조, 선택한 패턴과 그 이유. **"어떻게 설계되었고 왜 그런가"**에 답합니다.

예시: 폴더 구조, 레이어 패턴, 라이브러리 선택, 컴포넌트 다이어그램.

### development/

일상적인 개발 가이드. **"이 프로젝트에서 어떻게 작업하는가"**에 답합니다.

예시: 코드 규칙, 테스트 가이드, Git 워크플로, 로컬 설정, 린팅.

### operations/

코드를 프로덕션에 배포하고 실행을 유지하는 것과 관련된 모든 것. **"어떻게 배포하고 운영하는가"**에 답합니다.

예시: CI/CD 파이프라인, 환경 설정, 모니터링, 환경 변수.

## 폴더 추가 시점

세 가지 기본 폴더로 대부분의 프로젝트를 다룰 수 있습니다. 특정 도메인에서 프로젝트가 성장하면 전용 폴더를 추가합니다:

| 폴더 | 추가 시점 |
|------|-----------|
| `database/` | 프로젝트에 마이그레이션이 있는 자체 데이터베이스가 있는 경우 |
| `security/` | 인증 규칙, 권한 또는 컴플라이언스 문서화가 필요한 경우 |
| `integrations/` | 여러 외부 API 또는 서드파티 서비스가 있는 경우 |
| `api/` | 다른 사람이 사용하는 공개 API가 있는 경우 |

**규칙:** "혹시 몰라서" 빈 폴더를 만들지 마세요. 안에 넣을 실제 문서가 최소 하나 있을 때 생성하세요.

## 규칙

- Markdown으로 문서 작성, 파일당 하나의 주제
- 아키텍처 변경 시 최신 상태 유지
- `AGENTS.md`의 "상세 사양" 섹션에서 참조

## workplan/과의 차이

| | `wisdom/` | `workplan/` |
|---|-----------|-------------|
| **포함** | 영구적인 문서 | 생명주기가 있는 작업 계획 |

| **형식** | 자유 형식 | 헤더가 있는 표준화된 템플릿 |
| **목적** | 시스템 이해 | 해야 할 일 관리 |
