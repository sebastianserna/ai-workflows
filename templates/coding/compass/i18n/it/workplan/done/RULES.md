# Regole: done/

> Piani completati — archivio e riferimento.

## Cosa va qui

- Piani completamente implementati e verificati
- Questa cartella serve come registro storico

## Nomenclatura

**Con issue tracker (GitHub / GitLab):**

```
DONE-YYYY-MM-DD-iNNNN-descrizione.md
```

**Senza issue tracker:**

```
DONE-YYYY-MM-DD-descrizione.md
```

La data riflette quando il piano è stato completato.

## Regole chiave

- Ogni piano deve avere il YAML frontmatter con `state: "done"` (vedere `workplan/README.md` per il formato)
- Tutte le caselle di avanzamento devono essere spuntate
- I piani completati generalmente non vengono modificati
- Vedere `workplan/README.md` per il riferimento completo
