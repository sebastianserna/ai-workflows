# Regras: done/

> Planos concluídos — arquivo e referência.

## O que vai aqui

- Planos totalmente implementados e verificados
- Esta pasta serve como registro histórico

## Nomenclatura

**Com issue tracker (GitHub / GitLab):**

```
DONE-YYYY-MM-DD-iNNNN-description.md
```

**Sem issue tracker:**

```
DONE-YYYY-MM-DD-description.md
```

A data reflete quando o plano foi concluído.

## Regras-chave

- Cada plano deve ter YAML frontmatter com `state: "done"` (veja `workplan/README.md` para o formato)
- Todas as caixas de seleção de progresso devem estar marcadas
- Planos concluídos geralmente não devem ser modificados
- Veja `workplan/README.md` para a referência completa
