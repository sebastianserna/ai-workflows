# Regles : draft/

> Les brouillons sont une banque d'idees : ebauches de plans, decisions techniques, notes exploratoires.

## Ce que l'on trouve ici

- Idees en phase initiale pas encore pretes pour le backlog
- Enregistrements de decisions techniques
- Notes exploratoires et recherches

## Nommage

```
DRAFT-YYYY-MM-DD-description.md
```

Les brouillons ne portent **jamais** d'identifiant d'issue, quelle que soit la configuration du tracker.

## Regles cles

- Les brouillons ne suivent **pas** le workflow BACKLOG → CODING → DONE
- Quand un brouillon est suffisamment mur, il est promu en `backlog/` comme plan formel
- Voir `workplan/README.md` pour la reference complete

## Voir aussi

- `backlog/RULES.md` — Regles pour les plans en attente
- `coding/RULES.md` — Regles pour les plans en cours
- `done/RULES.md` — Regles pour les plans termines
