# Правила: draft/

> Черновики — это банк идей: наброски планов, технические решения, исследовательские заметки.

## Что здесь хранится

- Идеи на ранней стадии, ещё не готовые для backlog
- Записи технических решений
- Исследовательские заметки

## Именование

```
DRAFT-YYYY-MM-DD-description.md
```

Черновики **никогда** не содержат идентификатор задачи, независимо от настройки трекера.

## Ключевые правила

- Черновики **не** следуют процессу BACKLOG → CODING → DONE
- Когда черновик достаточно созревает, он продвигается в `backlog/` как формальный план
- Полную справку см. в `workplan/README.md`

## Смотрите также

- `backlog/RULES.md` — Правила для ожидающих планов
- `coding/RULES.md` — Правила для планов в работе
- `done/RULES.md` — Правила для завершённых планов
