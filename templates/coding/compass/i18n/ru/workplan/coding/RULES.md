# Правила: coding/

> Планы, над которыми активно ведётся работа.

## Что здесь хранится

- Планы, находящиеся в процессе выполнения
- Только планы, которые активно реализуются

## Именование

**С трекером задач (GitHub / GitLab):**

```
CODING-YYYY-MM-DD-iNNNN-description.md
```

**Без трекера задач:**

```
CODING-YYYY-MM-DD-description.md
```

Дата отражает момент начала работы.

## Ключевые правила

- Каждый план должен иметь YAML frontmatter с `state: "coding"` (формат см. в `workplan/README.md`)
- Обновляйте чекбоксы прогресса по мере выполнения шагов
- По завершении: переместите в `done/`, переименуйте префикс в `DONE`, обновите дату и frontmatter
- Для приостановки: верните в `backlog/`, переименуйте префикс в `BACKLOG`, обновите frontmatter
- Полную справку см. в `workplan/README.md`

## Смотрите также

- `backlog/RULES.md` — Правила для ожидающих планов
- `done/RULES.md` — Правила для завершённых планов
- `draft/RULES.md` — Правила для черновиков и идей
