# Правила: backlog/

> Ожидающие планы, готовые к работе.

## Что здесь хранится

- Планы, которые определены и готовы к началу (или ожидают приоритизации)
- Новые планы попадают сюда после создания

## Именование

**С трекером задач (GitHub / GitLab):**

```
BACKLOG-YYYY-MM-DD-iNNNN-description.md
```

**Без трекера задач:**

```
BACKLOG-YYYY-MM-DD-description.md
```

Дата отражает момент создания плана.

## Ключевые правила

- Каждый план должен иметь стандартизированный заголовок с `Статус: Backlog`
- **Если настроен трекер задач**, сначала создайте задачу (`gh issue create`, `glab issue create` и т.д.) и используйте номер задачи в имени файла (`iNNNN`)
- Для начала работы: переместите в `coding/`, переименуйте префикс в `CODING`, обновите дату и заголовок
- Планы также могут быть возвращены сюда из `coding/` при приостановке
- Полную справку см. в `workplan/README.md`

## Смотрите также

- `coding/RULES.md` — Правила для планов в работе
- `done/RULES.md` — Правила для завершённых планов
- `draft/RULES.md` — Правила для черновиков и идей
