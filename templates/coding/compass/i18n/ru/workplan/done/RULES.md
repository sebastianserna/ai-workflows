# Правила: done/

> Завершённые планы — архив и справка.

## Что здесь хранится

- Планы, которые полностью реализованы и проверены
- Эта папка служит историческим архивом

## Именование

**С трекером задач (GitHub / GitLab):**

```
DONE-YYYY-MM-DD-iNNNN-description.md
```

**Без трекера задач:**

```
DONE-YYYY-MM-DD-description.md
```

Дата отражает момент завершения плана.

## Ключевые правила

- Каждый план должен иметь YAML frontmatter с `state: "done"` (формат см. в `workplan/README.md`)
- Все чекбоксы прогресса должны быть отмечены
- Завершённые планы, как правило, не изменяются
- Полную справку см. в `workplan/README.md`
