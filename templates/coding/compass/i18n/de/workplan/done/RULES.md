# Regeln: done/

> Abgeschlossene Pläne — Archiv und Referenz.

## Was hier abgelegt wird

- Pläne, die vollständig implementiert und überprüft wurden
- Dieser Ordner dient als historisches Archiv

## Benennung

**Mit Issue-Tracker (GitHub / GitLab):**

```
DONE-YYYY-MM-DD-iNNNN-description.md
```

**Ohne Issue-Tracker:**

```
DONE-YYYY-MM-DD-description.md
```

Das Datum spiegelt den Zeitpunkt der Fertigstellung wider.

## Wichtige Regeln

- Jeder Plan muss den standardisierten Header mit `Status: Done` haben
- Alle Fortschritts-Checkboxen sollten angehakt sein
- Abgeschlossene Pläne sollten grundsätzlich nicht verändert werden
- Siehe `workplan/README.md` für die vollständige Referenz
