# ルール：draft/

> 下書きはアイデアバンク：計画の下書き、技術的決定事項、探索的メモ。

## このフォルダの内容

- まだbacklogに入れる準備ができていない初期段階のアイデア
- 技術的決定事項の記録
- 探索的メモと調査

## 命名規則

```
DRAFT-YYYY-MM-DD-description.md
```

トラッカーの設定に関わらず、下書きにはIssue識別子を**付けない**。

## 主要ルール

- 下書きはBACKLOG → CODING → DONEのワークフローに**従わない**
- 下書きが十分に成熟したら、`backlog/` に正式な計画として昇格させる
- 完全なリファレンスは `workplan/README.md` を参照

## 関連項目

- `backlog/RULES.md` — 未着手の計画のルール
- `coding/RULES.md` — 進行中の計画のルール
- `done/RULES.md` — 完了した計画のルール
