# 规则：draft/

> 草稿是一个想法库：计划草稿、技术决策、探索性笔记。

## 放置内容

- 尚未准备好进入 backlog 的早期想法
- 技术决策记录
- 探索性笔记和研究

## 命名

```
DRAFT-YYYY-MM-DD-description.md
```

无论跟踪器如何配置，草稿**永远不会**携带 Issue 标识符。

## 关键规则

- 草稿**不**遵循 BACKLOG → CODING → DONE 工作流
- 当草稿成熟时，将其提升到 `backlog/` 成为正式计划
- 参见 `workplan/README.md` 获取完整参考

## 另请参阅

- `backlog/RULES.md` — 待处理计划的规则
- `coding/RULES.md` — 进行中计划的规则
- `done/RULES.md` — 已完成计划的规则
