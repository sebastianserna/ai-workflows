# 规则：coding/

> 正在积极处理的计划。

## 放置内容

- 当前进行中的计划
- 仅限正在积极实施的计划

## 命名

**使用 Issue 跟踪器（GitHub / GitLab）：**

```
CODING-YYYY-MM-DD-iNNNN-description.md
```

**不使用 Issue 跟踪器：**

```
CODING-YYYY-MM-DD-description.md
```

日期反映工作开始的时间。

## 关键规则

- 每个计划必须有标准化标题，包含 `状态：Coding`
- 随着步骤完成，更新进度复选框
- 完成时：移动到 `done/`，将前缀重命名为 `DONE`，更新日期和标题
- 暂停时：移回 `backlog/`，将前缀重命名为 `BACKLOG`，更新标题
- 参见 `workplan/README.md` 获取完整参考

## 另请参阅

- `backlog/RULES.md` — 待处理计划的规则
- `done/RULES.md` — 已完成计划的规则
- `draft/RULES.md` — 草稿和想法的规则
