# 规则：done/

> 已完成的计划 — 归档和参考。

## 放置内容

- 已完全实施并验证的计划
- 此文件夹用作历史记录

## 命名

**使用 Issue 跟踪器（GitHub / GitLab）：**

```
DONE-YYYY-MM-DD-iNNNN-description.md
```

**不使用 Issue 跟踪器：**

```
DONE-YYYY-MM-DD-description.md
```

日期反映计划完成的时间。

## 关键规则

- 每个计划必须有 YAML frontmatter，包含 `state: "done"`（参见 `workplan/README.md` 了解格式）
- 所有进度复选框都应被勾选
- 已完成的计划通常不应被修改
- 参见 `workplan/README.md` 获取完整参考

## 另请参阅

- `backlog/RULES.md` — 待处理计划的规则
- `coding/RULES.md` — 进行中计划的规则
- `draft/RULES.md` — 草稿和想法的规则
